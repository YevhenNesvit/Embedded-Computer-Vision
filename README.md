# 👁️ Computer Vision & Edge AI Portfolio

Цей репозиторій є портфоліо моїх навчальних та практичних проектів у сфері комп'ютерного зору (Computer Vision) та штучного інтелекту. 
Головний фокус — обробка відео в реальному часі, розпізнавання об'єктів та інтеграція AI-алгоритмів з апаратними платформами (Edge AI / IoT / Robotics).

## 🛠 Технічний стек
* **Нейромережі (Deep Learning):** YOLOv8 (Ultralytics), Custom Model Training (Transfer Learning), PyTorch (CUDA Hardware Acceleration).
* **Трекінг та Алгоритми:** ByteTrack (SORT), Фільтр Калмана (Kalman Filter), Monocular Depth Estimation.
* **Обробка зображень:** OpenCV (Python), створення кастомних HUD-інтерфейсів, обробка MJPEG/MP4/UVC-потоків.
* **Web & Backend:** FastAPI, Uvicorn, Asynchronous Python, Multi-threading (рознесення інференсу та стрімінгу по різних потоках).
* **Мережа та Інтеграція:** HTTP/REST, взаємодія з мікроконтролерами (ESP32/STM32) через локальну мережу, Hardware-in-the-Loop (HIL) тестування.
* **Інструменти:** Python 3.12 (Miniconda), Linux (Ubuntu), VS Code.

## ⚡ Оптимізація продуктивності (Edge AI Optimization)

Одним із ключових завдань під час розробки було дослідження проблеми "вузького горлечка" (bottleneck) при запуску важких нейромереж на Edge-пристроях без використання GPU/NPU. Це критична вимога для автономної робототехніки та оборонних систем.

Базовий запуск моделі YOLOv8n (формат PyTorch, `FP32`) на процесорі показав значні затримки через високе навантаження при обчисленнях чисел з рухомою комою. Щоб адаптувати систему, було застосовано **Калібрування (Calibration)** та жорстку **Квантизацію (Quantization)** моделі до формату `INT8` за допомогою фреймворку **OpenVINO**.

### 📊 Результати бенчмарку (CPU Inference):

Тестування проводилось на процесорі архітектури x86 (AMD Ryzen 3 2200U, 2 Cores):

| Формат Моделі | Математика | Час обробки кадру (Latency) | Кадрів на секунду (FPS) | Навантаження на RAM |
| :--- | :--- | :--- | :--- | :--- |
| **PyTorch (.pt)** | `FP32` (Дроби) | ~350 мс | ~2.8 FPS | Високе (~1 ГБ, ризик OOM) |
| **OpenVINO** | `INT8` (Цілі) | **~155 - 200 мс** | **~5.0 - 6.4 FPS** | **Мінімальне** |

**Інженерні висновки:**
Конвертація моделі в `INT8` дозволила збільшити швидкість інференсу **майже в 2 рази** на тому ж самому апаратному забезпеченні, а також кардинально зменшити споживання оперативної пам'яті, усунувши випадки примусового завершення процесу системою (OOM Killer). 
*(Примітка: для розробки модуля ByteTrack та HUD система була переведена на апаратне прискорення CUDA (GTX 1050 Ti) для досягнення стабільних 30+ FPS з використанням багатопотоковості).*

### 🚀 GPU-Оптимізація (NVIDIA TensorRT & ONNX)

Для досягнення максимальної продуктивності на пристроях з графічними прискорювачами (GPU), базовий пайплайн інференсу на PyTorch був замінений на високооптимізовані рушії.

1. **ONNX (Open Neural Network Exchange):** Модель успішно експортована у формат `.onnx` (opset 17), що дозволило відв'язатися від важкого фреймворку PyTorch та підготувати архітектуру до кросплатформеного використання через `onnxruntime-gpu`.
2. **NVIDIA TensorRT (FP16):** Для цільового апаратного забезпечення (архітектура Pascal) скомпільовано нативний `.engine`. 
   * **Що було зроблено:** Зниження точності ваг з FP32 до FP16 (Half-precision), застосування **Layer Fusion** (злиття шарів) та фіксація графа обчислень (Memory Allocation Plan). 
   * **Результат:** Незважаючи на збільшення розміру файлу на диску (через вшитий мікрокод CUDA), споживання відеопам'яті (VRAM) під час інференсу суттєво зменшилось, а стабільність FPS зросла завдяки відсутності динамічного оверхеду PyTorch.

*(Додатково: проведено R&D у напрямку структурованого прунінгу (Structured Pruning) алгоритмами Magnitude/GroupNorm. Виявлено та проаналізовано проблему Dependency Hell у блоках C2f та Concat архітектури YOLOv8, що блокує графові аналізатори).*

## 📈 Плани на майбутнє (Defense Tech Roadmap)
Моя мета — поглибити практичні навички в Computer Vision та оптимізації продуктивності AI для застосування в автономних та охоронних системах. Що в планах:
* [x] **Model Optimization:** Конвертація моделей YOLO у формати OpenVINO/ONNX для підвищення FPS на процесорах (Квантизація в INT8).
* [x] **Custom Defense Datasets:** Fine-tuning (доопрацювання) моделей YOLO на специфічних датасетах (наприклад, розпізнавання БТР, танків та піхоти).
* [x] **Target Lock & Auto-Tracking:** Реалізація алгоритму автоматичного супроводження рухомої цілі по двох осях (X та Y) за допомогою STM32.
* [x] **Trajectory Prediction:** Впровадження Фільтра Калмана (ByteTrack / SORT) для прогнозування руху цілі при її тимчасовому зникненні з кадру.
* [x] **Depth Estimation:** Оцінка відстані до цілі за її розмірами в кадрі (Monocular Depth) з виводом на тактичний HUD.
* [x] **TensorRT Optimization:** Експорт та оптимізація моделей під відеокарти NVIDIA у формат .engine (Layer Fusion, FP16).
* [ ] **C++ Deployment:** Переписування інференс-пайплайну на C++ (ONNX Runtime / TensorRT C++ API) для розгортання на Edge-платах типу Jetson.
* [ ] **Sensor Fusion & Thermal Vision:** Дослідження можливостей роботи з тепловізійними (IR) камерами.
