# 👁️ Computer Vision & Edge AI Portfolio

Цей репозиторій є портфоліо моїх навчальних та практичних проектів у сфері комп'ютерного зору (Computer Vision) та штучного інтелекту. 
Головний фокус — обробка відео в реальному часі, розпізнавання об'єктів та інтеграція AI-алгоритмів з апаратними платформами (Edge AI / IoT).

## 🛠 Технічний стек
* **Нейромережі (Deep Learning):** YOLOv8 (Ultralytics), Object Detection.
* **Обробка зображень:** OpenCV (Python), обробка MJPEG-потоків.
* **Web & Backend:** FastAPI, Uvicorn, Asynchronous Python (для стрімінгу відео без затримок).
* **Мережа та Інтеграція:** HTTP/REST, взаємодія з мікроконтролерами (ESP32/STM32) через локальну мережу.
* **Інструменти:** Python 3.12 (Miniconda), Linux (Ubuntu), VS Code.

## ⚡ Оптимізація продуктивності (Edge AI Optimization)

Одним із ключових завдань під час розробки було дослідження проблеми "вузького горлечка" (bottleneck) при запуску важких нейромереж на пристроях з обмеженими обчислювальними ресурсами (Edge Devices) без використання GPU/NPU. Це критична вимога для автономної робототехніки та оборонних систем.

Базовий запуск моделі YOLOv8n (формат PyTorch, `FP32`) на процесорі показав значні затримки через високе навантаження при обчисленнях чисел з рухомою комою. Щоб адаптувати систему, було застосовано **Калібрування (Calibration)** та жорстку **Квантизацію (Quantization)** моделі до формату `INT8` за допомогою фреймворку **OpenVINO**.

### 📊 Результати бенчмарку (CPU Inference):



Тестування проводилось на процесорі архітектури x86 (AMD Ryzen 3 2200U, 2 Cores):

| Формат Моделі | Математика | Час обробки кадру (Latency) | Кадрів на секунду (FPS) | Навантаження на RAM |
| :--- | :--- | :--- | :--- | :--- |
| **PyTorch (.pt)** | `FP32` (Дроби) | ~350 мс | ~2.8 FPS | Високе (~1 ГБ, ризик OOM) |
| **OpenVINO** | `INT8` (Цілі) | **~155 - 200 мс** | **~5.0 - 6.4 FPS** | **Мінімальне** |

**Інженерні висновки:**
Конвертація моделі в `INT8` дозволила збільшити швидкість інференсу **майже в 2 рази** на тому ж самому апаратному забезпеченні, а також кардинально зменшити споживання оперативної пам'яті, усунувши випадки примусового завершення процесу системою (OOM Killer).

## 📈 Плани на майбутнє (Defense Tech Roadmap)
Моя мета — поглибити практичні навички в Computer Vision та оптимізації продуктивності AI для застосування в автономних та охоронних системах. Що в планах:
* [x] **Model Optimization:** Конвертація моделей YOLO у формати OpenVINO/ONNX для підвищення FPS на процесорах та Edge-пристроях (Квантизація в INT8).
* [ ] **Target Lock & Auto-Tracking:** Реалізація алгоритму автоматичного супроводження рухомої цілі (утримання в центрі кадру) за допомогою PID-регуляторів на базі STM32.
* [ ] **Custom Defense Datasets:** Fine-tuning (доопрацювання) моделей YOLO на специфічних датасетах (наприклад, детекція БПЛА в небі, розпізнавання типів транспорту).
* [ ] **Sensor Fusion & Thermal Vision:** Дослідження можливостей роботи з тепловізійними (IR) або нічними камерами для розпізнавання об'єктів в умовах нульової видимості.
* [ ] **Depth Estimation:** Оцінка відстані до цілі за її розмірами в кадрі (Bounding Box) для передачі координат.
